Pipeline

  1. Read Pricelist 2025.xlsx from Drive (file id 1AbjgiZdjB2VTHKiJw0paVQQsjAWlvroC). Parse section-headed sheet — each family carries a wholesale discount.
  2. Use the Net price 2025 column as EXW (NOK).
  3. Convert NOK → EUR via open.er-api.com (ECB-backed, no key). Fallback chain: frankfurter.app → 0.087 hardcoded.
  4. Feed EUR FOB + family + (no per-SKU CBM) into shared/landed_pricing.py: shipping-automation estimate_landed_cost on the Europe → Laem Chabang LCL route, Baltic-rate calibrated.
  5. Tiered logistics clamp (80 / 60 / 45 / 35 % floor by FOB band).
  6. Retail THB = landed / (1 − 0.40); USD & EUR derived at live FX.
  7. Write to Firestore vendors/rampline/pricelists/2026-05-13 — one audit doc, variants map keyed by sanitized article code.

Deferred Medusa variant push

Rampline's 54 Medusa products each carry a single Default variant keyed on the WooCommerce numeric ID (e.g. 26371). The pricelist's 127 article codes (RB35, BP 34 LF, …) are variant-level SKUs that don't yet exist in Medusa. Creating per-article variants is a separate migration; three families (SHOCKDECK, climbing pole, balance arch) appear to have no matching Medusa product either.

For now we audit landed + retail in Firestore only. The CSV at rampline-catalog/data/pricelist_2026-05-13_landed.csv is the source of truth for any downstream pricing decision.
