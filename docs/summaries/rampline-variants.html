Variant metadata (audit fields)

Each new variant carries this metadata for downstream reconciliation with the Firestore audit doc vendors/rampline/pricelists/2026-05-13 and the source xlsx:

  • article_code — original pricelist code (e.g. RB35 AG)
  • family — pricelist family name
  • family_discount — wholesale discount fraction
  • net_nok, recommended_nok — pricelist amounts
  • description — pricelist row description
  • pricelist_date — 2026-05-13
  • source — rampline-pricelist-2025

No prices on variants — pricing handled separately via the Firestore-backed pricing-config flow (v2.20.1) and a follow-up sync script. New size sub-products are status=draft until prices land.
